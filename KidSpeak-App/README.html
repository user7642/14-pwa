📘 KidSpeak Project Blueprint

Dự án PWA giáo dục đa ngôn ngữ sử dụng cơ chế lưu trữ Offline (OPFS).

1. Luồng hoạt động (Execution Flow)

01
Khởi tạo: app.js đọc data.json ở thư mục gốc để biết danh sách các chủ đề (Topics) và Link tải ZIP từ GitHub.
02
Kiểm tra Media: App check file manifest.json trong folder media/.
• Dev: Đọc trực tiếp qua Symlink.
• Prod: Check trong bộ nhớ trình duyệt (OPFS).
03
Tải & Giải nén: Nếu chưa có dữ liệu, downloader.js tải ZIP, dùng JSZip xả nén và storage.js ghi vào OPFS.
04
Tương tác: Người dùng chọn ngôn ngữ (VN/EN) -> app.js thay đổi biến currentLang -> Map đường dẫn ảnh/âm thanh tương ứng để hiển thị.

2. Cấu trúc thư mục quan trọng

KidSpeak-App/
├── data.json           <-- "Bản đồ tổng" chứa danh sách Topic & Link ZIP
├── index.html          <-- Giao diện chính (Accordion)
├── media/              <-- Symlink trỏ tới Media/topics/ (Dùng khi Dev)
├── js/
│   ├── app.js          <-- Điều khiển chính (Logic UI & Ngôn ngữ)
│   ├── storage.js      <-- Xử lý lưu file vào bộ nhớ trình duyệt
│   └── downloader.js   <-- Tải và giải nén ZIP
└── Media/
    ├── topics/         <-- Dữ liệu gốc (Ảnh, MP3)
    └── build-zip.sh    <-- Script nén ZIP chuẩn để upload GitHub

3. Lưu ý khi di chuyển sang dự án khác

  • Đồng nhất ID: ID trong data.json phải trùng tên folder chứa Media.
  • Cấu trúc ZIP: Khi nén ZIP, không được để folder cha bao ngoài. File manifest.json phải nằm ngay ở root của file ZIP.
  • CORS: Link tải ZIP (GitHub Releases) phải cho phép tải từ bên ngoài (thường GitHub đã hỗ trợ sẵn).
  • Symlink: Trên Arch Linux, dùng ln -s để tạo liên kết ảo giúp preview ảnh ngay lập tức mà không cần chờ nén/giải nén.
