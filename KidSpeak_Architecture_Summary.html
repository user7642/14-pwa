Tài liệu Kiến trúc Dự án: KidSpeak

Cập nhật lần cuối: 13/05/2026 - Mục tiêu: Tài liệu hóa để nắm bắt nhanh sau thời gian dài không tiếp xúc.

1. Tổng quan dự án

KidSpeak là một Progressive Web App (PWA) hỗ trợ giáo dục cho trẻ em, cung cấp nội dung đa phương tiện (âm thanh, hình ảnh) theo chủ đề. Dự án tập trung vào tính năng Offline First, cho phép phụ huynh tải và sử dụng tài nguyên mà không cần internet thường xuyên.

  • HTML5/CSS3
  • Vanilla JavaScript
  • Service Worker
  • PWA (Web Manifest)
  • JSON-based Data

2. Cấu trúc thư mục (File System)

/
├── index.html          # Giao diện chính (Accordion, Layout)
├── manifest.json       # Cấu hình PWA
├── sw.js               # Service Worker (Chiến lược Cache First)
├── data.json           # Dữ liệu chủ đề, file ZIP, kích thước
├── css/
│   └── style.css       # Style chính (đã tối ưu căn giữa & responsive)
└── js/
    ├── app.js          # Logic khởi tạo và quản lý chung
    ├── storage.js      # Xử lý lưu trữ IndexedDB/LocalStorage
    ├── downloader.js   # Logic tải và giải nén tệp ZIP
    └── ui-manager.js   # Xử lý render giao diện động

3. Chiến lược Cache & Offline (sw.js)

Dự án sử dụng chiến lược Cache First:

  • Install Event: Cache các file "core" (CSS, JS, Logo).
  • Fetch Event:
    • Ưu tiên trả về từ Cache. Nếu không có mới thực hiện fetch.
    • Hỗ trợ status: 0 (Opaque responses) cho tài nguyên từ domain khác.
    • Bỏ qua các URL không phải HTTP (như chrome-extension).

4. Các thành phần quan trọng đã tối ưu

Thành phần Mô tả / Lưu ý quan trọng
Dòng hướng dẫn Sử dụng width: fit-content và margin: auto để căn giữa tuyệt đối 1 dòng.
Dữ liệu (data.json) Trường size hiện là metadata (chưa tham gia logic hiển thị nhưng dùng để dự báo dung lượng).
Service Worker Đã fix lỗi chặn các URL chrome-extension và hỗ trợ lưu tài nguyên ngoại vi.

5. Quy trình thêm nội dung mới

  1. Cập nhật file ZIP vào thư mục assets/zips/.
  2. Khai báo thông tin mới (id, title, size, url) vào data.json.
  3. Nếu có thay đổi quan trọng về file hệ thống, tăng version CACHE_NAME trong sw.js để ứng dụng cập nhật.
